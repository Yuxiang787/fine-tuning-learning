# 超参数调优指南

## 核心超参数

### 1. 学习率 (Learning Rate)

**影响**: 最重要的超参数，决定模型更新步长。

| 微调方式 | 推荐范围 | 起点 |
|----------|----------|------|
| LoRA | 1e-4 ~ 5e-4 | 2e-4 |
| 全量微调 | 1e-5 ~ 5e-5 | 2e-5 |
| QLoRA | 1e-4 ~ 2e-4 | 1e-4 |

**调优策略**:
```
1. 从 2e-4 (LoRA) 或 2e-5 (全量) 开始
2. 训练 loss 不降 → 增大学习率
3. 训练不稳定/震荡 → 减小学习率
4. 使用学习率调度器（cosine, linear）
```

**迹象判断**:
- **太大**: loss 震荡、发散、生成乱码
- **太小**: loss 下降慢、欠拟合
- **合适**: loss 平稳下降

### 2. LoRA 秩 (r)

**影响**: 控制可训练参数量和表达能力。

| r 值 | 参数量占比 | 适用场景 |
|------|------------|----------|
| 4 | ~0.1% | 简单任务，极少数据 |
| 8 | ~0.5% | 大多数任务（推荐起点） |
| 16 | ~1% | 复杂任务，数据充足 |
| 32 | ~2% | 特殊需求 |
| 64+ | ~4% | 罕见情况 |

**调优策略**:
```
1. 从 r=8 开始
2. 效果不足 → r=16
3. 继续增加收益递减
4. 大模型用小 r，小模型用大 r
```

### 3. LoRA Alpha (α)

**影响**: 缩放 LoRA 输出的权重。

**常见设置**:
```yaml
# 方案 1: alpha = 2r (推荐)
r=8, alpha=16
r=16, alpha=32

# 方案 2: alpha = r (保守)
r=8, alpha=8

# 方案 3: 固定大值
r=8, alpha=32
```

### 4. Batch Size

**影响**: 梯度估计的稳定性和显存占用。

| 显存 | 推荐 batch size | 梯度累积 |
|------|-----------------|----------|
| 4GB | 1 | 8-16 |
| 8GB | 2-4 | 4-8 |
| 16GB | 4-8 | 2-4 |
| 24GB+ | 8-16 | 1-2 |

**有效 batch size**:
```
effective_batch = batch_size × gradient_accumulation_steps

推荐有效 batch: 8-32
```

### 5. 训练轮数 (Epochs)

**影响**: 模型看到数据的次数。

| 数据量 | 推荐 epochs |
|--------|-------------|
| <100 | 10-20 |
| 100-500 | 5-10 |
| 500-2000 | 3-5 |
| 2000+ | 2-3 |

**过拟合迹象**:
- 训练 loss 持续下降，验证 loss 上升
- 生成结果过度拟合训练数据

## 调优流程

### 第一阶段：快速验证

```yaml
# 目标：快速验证流程可行
lora_r: 8
lora_alpha: 16
learning_rate: 2e-4
batch_size: 4
epochs: 1
```

**时间**: 10-30 分钟
**目标**: 确认无错误，loss 下降

### 第二阶段：学习率搜索

```yaml
# 固定其他参数，搜索学习率
试验 1: lr = 1e-4
试验 2: lr = 2e-4  ← 推荐起点
试验 3: lr = 5e-4
```

**选择**: 验证 loss 最低的学习率

### 第三阶段：LoRA 秩调整

```yaml
# 固定最佳学习率
试验 1: r = 8
试验 2: r = 16
试验 3: r = 32
```

**选择**: 验证集效果最好的 r

### 第四阶段：精细调优

```yaml
# 微调其他参数
lora_dropout: 0.05 ~ 0.2
warmup_ratio: 0.03 ~ 0.1
weight_decay: 0.01 ~ 0.1
```

## 常见问题诊断

### 问题 1: Loss 不下降

**可能原因**:
- 学习率太小
- 模型容量不足
- 数据质量问题

**解决方案**:
```
1. 增大学习率 (×2 或 ×5)
2. 增加 LoRA 秩 (8→16)
3. 检查数据格式和质量
4. 增加训练轮数
```

### 问题 2: Loss 震荡/发散

**可能原因**:
- 学习率太大
- Batch size 太小
- 数据噪声大

**解决方案**:
```
1. 减小学习率 (÷2 或 ÷5)
2. 增大 batch size 或梯度累积
3. 清洗数据
4. 增加 warmup
```

### 问题 3: 过拟合

**迹象**:
- 训练 loss 很低，验证 loss 高
- 生成结果重复训练数据

**解决方案**:
```
1. 减少训练轮数
2. 增加 dropout (0.1→0.2)
3. 增加数据量
4. 减小 LoRA 秩
5. 增加 weight decay
```

### 问题 4: 欠拟合

**迹象**:
- 训练 loss 和验证 loss 都高
- 生成结果不相关

**解决方案**:
```
1. 增加训练轮数
2. 增大学习率
3. 增加 LoRA 秩
4. 检查提示模板
```

### 问题 5: 显存不足 (OOM)

**解决方案**:
```yaml
# 优先级从低到高
1. 减小 batch_size: 4→2→1
2. 增加 gradient_accumulation_steps
3. 减小 max_length: 512→256
4. 启用梯度检查点
5. 使用 QLoRA (4 位量化)
```

## 推荐配置

### 配置 1: 快速入门

```yaml
# 适合：快速验证，资源有限
model: facebook/opt-125m
lora_r: 8
lora_alpha: 16
learning_rate: 2e-4
batch_size: 4
epochs: 3
max_length: 256
```

### 配置 2: 标准配置

```yaml
# 适合：大多数任务，效果优先
model: Qwen/Qwen2.5-0.5B
lora_r: 16
lora_alpha: 32
learning_rate: 2e-4
batch_size: 4
epochs: 5
max_length: 512
warmup_ratio: 0.05
```

```yaml

```

### 配置 4: 全量微调

```yaml

full_finetune: true
learning_rate: 2e-5
batch_size: 2
epochs: 3
weight_decay: 0.01
max_grad_norm: 1.0
```

## 学习率调度器

### 类型选择

```python
# Cosine (推荐)
lr_scheduler = "cosine"

# Linear with warmup
lr_scheduler = "linear"
warmup_ratio = 0.1

# Constant
lr_scheduler = "constant"
```

### Warmup 设置

```yaml
# 推荐：3-10% 的总步数作为 warmup
warmup_ratio: 0.05  # 5%

# 或指定步数
warmup_steps: 100
```

## 实验管理

### 记录实验

```markdown
| 实验 | lr | r | alpha | epochs | 验证 loss | 备注 |
|------|-----|---|-------|--------|----------|------|
| 1 | 1e-4 | 8 | 16 | 3 | 1.23 | 基线 |
| 2 | 2e-4 | 8 | 16 | 3 | 1.15 | 更好 |
| 3 | 2e-4 | 16 | 32 | 3 | 1.08 | 最佳 |
| 4 | 2e-4 | 16 | 32 | 5 | 1.12 | 过拟合 |
```

### 最佳实践

1. **一次只改一个参数**
2. **记录所有实验**
3. **使用验证集选择**
4. **保留最佳检查点**
