# 常见问题 FAQ

## 入门问题

### Q1: 我需要什么样的硬件？

**最低要求**:
- CPU: 可以运行，但很慢（适合测试流程）
- 内存：8GB+
- 存储：10GB+

**推荐配置**:
- GPU: NVIDIA GPU，4GB+ 显存
- 内存：16GB+
- 存储：20GB+ SSD

**云资源**:
- Google Colab (免费 T4 GPU)
- Kaggle Notebook (免费 P100 GPU)
- AutoDL/恒源云 ( affordable GPU 租赁)

### Q2: 我应该从哪个模型开始？

**初学者推荐**:

| 模型 | 大小 | 特点 | 适用场景 |
|------|------|------|----------|
| OPT-125M | 125M | 最小，最快 | 学习、测试 |
| Qwen2.5-0.5B | 0.5B | 中文友好 | 中文任务 |
| TinyLlama-1.1B | 1.1B | 能力强 | 进阶使用 |

**不建议一开始就使用 7B+ 大模型**。

### Q3: 我需要多少数据？

**最少**: 20-50 条（验证流程）
**推荐**: 100-500 条（有效微调）
**理想**: 1000+ 条（良好效果）

**数据质量 > 数量**，100 条高质量数据胜过 1000 条噪声数据。

### Q4: 训练需要多长时间？

**OPT-125M, 100 条数据，3 epochs**:
- CPU: 30-60 分钟
- GPU (T4): 5-10 分钟

**Qwen-0.5B, 500 条数据，3 epochs**:
- GPU (T4): 15-30 分钟

## 安装问题

### Q5: 依赖安装失败

**问题**: `pip install` 报错

**解决**:
```bash
# 1. 升级 pip
pip install --upgrade pip

# 2. 使用国内镜像
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 3. 逐个安装
pip install torch
pip install transformers
pip install peft
pip install datasets
```

### Q6: CUDA 版本不匹配

**问题**: `CUDA error`, `cuDNN error`

**解决**:
```bash
# 检查 CUDA 版本
nvidia-smi

# 安装对应版本的 torch
# CUDA 11.8
pip install torch --index-url https://download.pytorch.org/whl/cu118

# CUDA 12.1
pip install torch --index-url https://download.pytorch.org/whl/cu121
```

### Q7: 导入错误

**问题**: `ModuleNotFoundError: No module named 'xxx'`

**解决**:
```bash
# 确保在正确的环境中
which python
pip list | grep transformers

# 重新安装缺失的包
pip install xxx
```

## 训练问题

### Q8: 显存不足 (OOM)

**错误**: `CUDA out of memory`

**解决**（优先级从高到低）:
```yaml
1. 减小 batch_size: 4 → 2 → 1
2. 增加 gradient_accumulation_steps: 2 → 4 → 8
3. 减小 max_length: 512 → 256 → 128
4. 使用更小的模型
5. 使用 QLoRA (4 位量化)
```

### Q9: Loss 不下降

**可能原因**:
1. 学习率太小
2. 数据格式错误
3. 模型没有正确加载

**检查清单**:
```
[ ] 确认数据格式正确（instruction/input/output）
[ ] 确认 tokenizer 和模型匹配
[ ] 尝试增大学习率
[ ] 检查训练输出，确认有梯度更新
```

### Q10: 训练很慢

**加速方法**:
```yaml
# 1. 使用更小的模型
model_name: "facebook/opt-125m"  # 而不是 1B+

# 2. 减小数据长度
max_length: 256  # 而不是 512+

# 3. 启用混合精度
fp16: true

# 4. 使用 GPU
# 确保 CUDA 可用
```

### Q11: 生成的内容重复训练数据

**原因**: 过拟合

**解决**:
```yaml
1. 减少训练轮数：5 → 3
2. 增加 dropout: 0.1 → 0.2
3. 增加数据量
4. 减小 LoRA 秩：16 → 8
```

## 效果问题

### Q12: 微调后效果没提升

**可能原因**:
1. 数据量太少
2. 任务太复杂
3. 模型容量不足

**建议**:
```
1. 增加数据量到 500+ 条
2. 检查数据质量
3. 尝试更大的模型
4. 调整超参数
```

### Q13: 中文效果不好

**解决**:
```yaml
# 1. 使用中文友好的模型
model_name: "Qwen/Qwen2.5-0.5B"  # 而不是 OPT

# 2. 确保数据是 UTF-8 编码
# 3. 检查 tokenizer 是否正确处理中文
```

### Q14: 模型不遵循指令

**可能原因**:
1. 提示模板不对
2. 训练时和推理时格式不一致

**检查**:
```python
# 确保训练和推理使用相同的模板
训练：### Instruction:\n{instruction}\n\n### Output:\n{output}
推理：### Instruction:\n{instruction}\n\n### Output:\n
```

## 部署问题

### Q15: 如何保存和加载模型？

**保存**:
```python
# LoRA 模型
model.save_pretrained("./output")
tokenizer.save_pretrained("./output")
```

**加载**:
```python
from peft import PeftModel
from transformers import AutoTokenizer, AutoModelForCausalLM

# 加载基础模型
base_model = AutoModelForCausalLM.from_pretrained("facebook/opt-125m")
tokenizer = AutoTokenizer.from_pretrained("facebook/opt-125m")

# 加载适配器
model = PeftModel.from_pretrained(base_model, "./output")
```

### Q16: 如何合并 LoRA 权重？

```python
from peft import PeftModel

# 加载
model = PeftModel.from_pretrained(base_model, "./output")

# 合并
merged_model = model.merge_and_unload()

# 保存完整模型
merged_model.save_pretrained("./merged_output")
```

### Q17: 如何在生产环境使用？

**推荐方案**:
```python
# 1. 合并权重
merged_model = model.merge_and_unload()

# 2. 导出为 ONNX (可选)
from optimum.onnxruntime import ORTModelForCausalLM

# 3. 使用推理优化
# - vLLM
# - Text Generation Inference
# - LMStudio
```

## 快速诊断流程

```
问题 → 检查清单

训练失败:
[ ] 显存是否足够？
[ ] 数据格式是否正确？
[ ] 模型是否成功加载？

效果不好:
[ ] 数据量是否足够？
[ ] 超参数是否合理？
[ ] 训练轮数是否合适？

推理错误:
[ ] 模型路径是否正确？
[ ] 提示模板是否一致？
[ ] 设备是否可用？
```
