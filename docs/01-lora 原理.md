# LoRA 原理详解

## 什么是 LoRA？

LoRA（Low-Rank Adaptation，低秩自适应）是一种**参数高效微调**技术，由微软研究院在 2021 年提出。

核心思想：**在冻结预训练模型权重的情况下，通过添加低秩矩阵来实现微调。**

## 传统微调 vs LoRA

### 传统全量微调

```
预训练权重 W₀ (d×k) → 微调 → 新权重 W' (d×k)

问题：
- 需要存储完整的模型副本
- 大模型（7B+）需要数十 GB 显存
- 每个任务需要一个完整模型
```

### LoRA 微调

```
预训练权重 W₀ (d×k) 冻结

添加低秩适配器：
    ΔW = B × A
    其中：B (d×r), A (r×k), r << min(d,k)

前向传播：
    h = W₀x + ΔWx = W₀x + BAx

优势：
- 只需训练 A 和 B 矩阵
- 参数量减少 100-1000 倍
- 每个任务只需存储轻量适配器
```

## 数学原理

### 低秩分解

假设原始权重矩阵 W ∈ ℝ^(d×k)，微调后的变化为 ΔW。

LoRA 假设 ΔW 是**低秩**的，可以分解为：

```
ΔW = B × A

其中：
- B ∈ ℝ^(d×r)
- A ∈ ℝ^(r×k)
- r << min(d, k)  （秩远小于原始维度）
```

### 参数量对比

以 OPT-125M 的 attention 层为例：

```
原始权重：d=768, k=768
参数量 = 768 × 768 = 589,824

LoRA (r=8):
- A 矩阵：768 × 8 = 6,144
- B 矩阵：8 × 768 = 6,144
总参数量 = 12,288

减少倍数 = 589,824 / 12,288 ≈ 48 倍
```

### 缩放系数

LoRA 的实际输出会乘以缩放系数：

```
输出 = (α/r) × BAx

其中：
- α 是超参数（通常设为 16 或 32）
- r 是 LoRA 秩

作用：稳定训练，类似于学习率的作用
```

## LoRA 架构

```
                    输入 x
                      │
        ┌─────────────┴─────────────┐
        │                           │
        ▼                           ▼
   ┌─────────┐               ┌──────────┐
   │  W₀     │               │  LoRA    │
   │ (冻结)  │               │  B×A     │
   └────┬────┘               └────┬─────┘
        │                         │
        │    W₀x                  │    BAx
        │         ┌───────────────┘
        ▼         ▼
      ┌─────────────┐
      │   相加      │  →  输出 h = W₀x + BAx
      └─────────────┘
```

### 应用到 Transformer

LoRA 通常应用于 Transformer 的以下模块：

```python
# 注意力机制
q_proj: W_q → W_q + ΔW_q
k_proj: W_k → W_k + ΔW_k
v_proj: W_v → W_v + ΔW_v
o_proj: W_o → W_o + ΔW_o

# FFN 层（可选）
gate_proj: W_gate → W_gate + ΔW_gate
up_proj:   W_up   → W_up   + ΔW_up
down_proj: W_down → W_down + ΔW_down
```

### 目标模块选择

```python
# 仅注意力（最常用）
target_modules = ["q_proj", "v_proj"]

# 全部注意力
target_modules = ["q_proj", "k_proj", "v_proj", "o_proj"]

# 注意力 + FFN（更强，但参数更多）
target_modules = ["q_proj", "k_proj", "v_proj", "o_proj",
                  "gate_proj", "up_proj", "down_proj"]

# 自动选择（peft 库）
target_modules = None  # 根据模型架构自动推断
```

## 超参数详解

### 秩 r

**定义**：LoRA 矩阵的秩，控制表达能力。

| r 值 | 参数量 | 表达能力 | 推荐场景 |
|------|--------|----------|----------|
| 4 | 最少 | 弱 | 简单任务，极少数据 |
| 8 | 少 | 中等 | 大多数任务（推荐起点） |
| 16 | 中等 | 强 | 复杂任务，数据充足 |
| 32+ | 多 | 很强 | 特殊需求，大量数据 |

**经验法则**：
- 从 r=8 开始
- 效果不足时增加到 16
- 继续增加通常收益递减

### Alpha (α)

**定义**：缩放系数，控制 LoRA 输出的权重。

**常见设置**：
- α = 2r（最常用，如 r=8, α=16）
- α = r（保守）
- α = 32 或 64（固定值，不随 r 变化）

**作用**：
- 较大的 α：LoRA 贡献更大
- 较小的 α：更接近原始模型

### Dropout

**定义**：防止过拟合的正则化技术。

**推荐值**：
- 小数据集（<1000）：0.1-0.2
- 中等数据集：0.05-0.1
- 大数据集：0.0-0.05

## 为什么 LoRA 有效？

### 1. 过参数化假设

预训练模型已经学到了丰富的表示，微调只需要**小幅调整**。低秩矩阵足以捕捉这些调整。

### 2. 梯度流动

冻结的主干网络提供稳定的梯度，LoRA 模块学习残差更新。

### 3. 隐式正则化

低秩约束相当于一种正则化，减少过拟合风险。

## 变体与扩展

### QLoRA（Quantized LoRA）

**核心**：4 位量化 + LoRA

```
预训练权重 → 4 位量化 → 冻结
LoRA 适配器 → FP16 → 可训练

优势：
- 7B 模型仅需 5GB 显存
- 速度接近 FP16 微调
```

### LoRA+

**改进**：为 A 和 B 矩阵使用不同的学习率。

```python
# 建议配置
loraplus_lr_ratio = 16  # B 矩阵的学习率是 A 的 16 倍
```

### AdaLoRA

**改进**：自适应秩分配。

```
重要层 → 高秩
次要层 → 低秩
```

## 实战建议

### 第一次微调

```yaml
lora_r: 8
lora_alpha: 16
lora_dropout: 0.1
target_modules: ["q_proj", "v_proj"]
learning_rate: 2e-4
num_epochs: 3
```

### 效果不佳时

1. **增加数据**：至少 100 条高质量样本
2. **增加 r**：从 8 增加到 16 或 32
3. **调整学习率**：尝试 1e-4 到 5e-4
4. **增加轮数**：5-10 轮（注意过拟合）
5. **检查数据质量**：格式一致，无噪声

### 资源受限

```yaml
# 使用 QLoRA
load_in_4bit: true
lora_r: 16
batch_size: 1
gradient_accumulation_steps: 8
```
